## Agent Brief

**Category:** enhancement
**Summary:** Add static scenario obstacle geometry to Offline Web replay data and render it as real 3D geometry in the passive viewer.

**Current behavior:**
Offline Web replay scene data exposes obstacle samples as points. The passive viewer renders those samples with a point material, so narrow gaps and walls appear as isolated dots rather than physical 3D obstacle bodies.

**Desired behavior:**
Scenario replay data should be able to include static obstacle visualization geometry, and the viewer should render that geometry as solid or translucent 3D objects. The existing obstacle point cloud should remain available because it is the controller-facing observation/collision input and is still useful for debugging.

**Key interfaces:**

- Replay scene schema — add a stable, optional static obstacle geometry field separate from `obstacle_points`.
- Scenario obstacle configuration — support enough geometry metadata to represent the existing static scenario obstacles as boxes or equivalent simple volumes.
- Passive Web 3D viewer layer controls — distinguish obstacle geometry from obstacle points, or make it clear when both are shown.
- Scenario runner/exporter — preserve world-coordinate replay data and do not require the viewer to reconstruct controller-local observations.

**Acceptance criteria:**

- [x] `narrow_gap_t_volume_3d` replay data includes static obstacle geometry sufficient to render the gap walls as 3D bodies.
- [x] The viewer renders obstacle geometry as 3D objects, not only as points.
- [x] The existing `obstacle_points` export remains present and usable for debugging/controller inspection.
- [x] Geometry rendering is world-frame and passive; the browser does not own simulation or collision logic.
- [x] Tests cover replay export of obstacle geometry for at least one scenario and a viewer smoke/static check for the geometry rendering hook.

**Out of scope:**

- Replacing the controller's `3D obstacle points` input contract.
- Adding a full mesh, ESDF, occupancy grid, or browser-side collision system.
- Dynamic obstacle geometry.
