## Agent Brief

**Category:** enhancement
**Summary:** Improve playback smoothness with display-only interpolation and make it clear how animation smoothness differs from controller smoothness telemetry.

**Current behavior:**
The viewer steps through exported replay frames discretely during playback. For a 40-frame run with simulation steps around 0.15 seconds, the robot can appear jumpy even when the underlying trajectory telemetry is acceptable. The viewer shows telemetry values, but it does not explain or expose frame cadence enough to distinguish playback sampling from algorithmic smoothness.

**Desired behavior:**
The viewer should support display-only interpolation between replay frames for smoother playback while preserving exact frame scrubbing. It should also expose enough cadence/telemetry context that users can tell whether rough motion is due to visualization frame count or controller behavior.

**Key interfaces:**

- Passive Web 3D viewer playback loop — interpolate rendered robot pose and path cursor between adjacent replay frames during continuous playback.
- Timeline scrubbing and single-frame inspection — remain exact and frame-index based.
- Replay metadata or viewer metrics — show frame count, playback frame index, and relevant smoothness telemetry without claiming interpolation improves controller behavior.
- Scenario runner/exporter — do not change MPPI control, state integration, or telemetry calculations as part of this viewer issue.

**Acceptance criteria:**

- [x] Continuous playback uses display-only interpolation for robot pose between adjacent replay frames.
- [x] Timeline scrubbing still lands on exact exported frames and displays exact frame metrics.
- [x] The viewer exposes replay frame count/cadence and current smoothness telemetry clearly enough to distinguish animation smoothness from controller smoothness.
- [x] Interpolation does not alter exported replay JSON, scenario summaries, or smoothness telemetry.
- [x] A test or documented manual check covers playback with interpolation enabled on `narrow_gap_t_volume_3d`.

**Out of scope:**

- MPPI parameter tuning, critic redesign, command smoothing, or cost-function changes.
- Increasing simulation step frequency before viewer interpolation/diagnostics are in place.
- Treating Web animation smoothness as proof of controller smoothness.
