# Remove or gate the yellow heading cone on the T-shaped robot

Status: implemented
Category: bug

## Parent

.scratch/3d-web-replay-and-scenario-suite/PRD.md

## Triage Notes

**What we've established so far:**

- The yellow triangle is an intentional viewer heading marker implemented as a cone mesh, not part of the T-shaped 3D robot volume.
- The PRD requires the T-shaped robot volume to be rendered from the authoritative collision geometry and rejects decorative robot models that do not match collision geometry.
- The T-shaped body itself should make yaw inspectable, so a yellow cone attached to the robot can mislead users into thinking it is part of the collision volume.
- No matching `.out-of-scope/` rejection exists in this repo.

## Agent Brief

**Category:** bug
**Summary:** Remove the yellow heading cone from the default T-shaped robot rendering, or make it an explicitly optional visual aid.

**Current behavior:**
The passive viewer renders the T-shaped robot volume from box-union geometry and also adds a yellow cone/triangle heading marker at the front. Users can mistake this marker for part of the robot body or collision volume.

**Desired behavior:**
By default, the robot rendering should show only the authoritative T-shaped 3D robot volume and any non-misleading edge/pose styling. If a heading aid is retained, it must be opt-in or clearly separated from the robot volume layer so it cannot be confused with collision geometry.

**Key interfaces:**

- Passive Web 3D viewer robot rendering — body geometry should remain derived from replay `robot_volume`.
- Viewer layer controls or visual settings — any retained heading aid should have an explicit toggle separate from the robot volume.
- Viewer tests — should assert the robot body is rendered from volume config without requiring a decorative cone marker.

**Acceptance criteria:**

- [x] Loading a replay no longer shows a yellow cone/triangle as part of the default T-shaped robot.
- [x] The T-shaped body orientation remains understandable during playback.
- [x] If a heading marker remains available, it is disabled by default and controlled separately from the robot volume layer.
- [x] Tests no longer require `ConeGeometry` as proof of heading clarity; they verify authoritative robot-volume rendering instead.

**Out of scope:**

- Changing the T-shaped 3D robot volume used for collision evaluation.
- Adding decorative or mesh-based robot models.
- Changing yaw-only 3D state semantics.
